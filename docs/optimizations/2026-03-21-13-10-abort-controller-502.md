# 优化记录：AbortController 导致 502 错误

**时间**: 2026-03-21 13:10
**触发**: 用户报告点击 Start Conversation 后 502 错误

## 问题

点击开始对话后页面显示 502 错误。服务器日志：
```
"request ended without sending any chunks"
Error: failed to pipe response
```

## 根因

Sprint 1 中添加的 LLM 请求超时机制使用了 `AbortController`:

```typescript
const controller = new AbortController();
const timeout = setTimeout(() => controller.abort(), 60_000);
rawStream = await this.client.messages.stream({
  ...createParams,
  signal: controller.signal,  // ← 问题所在
} as unknown as Anthropic.MessageStreamParams);
clearTimeout(timeout); // ← 在 finally 中立即清除
```

两个问题叠加：
1. `signal` 通过 `as unknown as` 强制类型转换传入，与自定义代理 baseURL (`https://crs.chenge.ink/api`) 不兼容
2. `finally` 块在 `stream()` 对象创建后立即清除 timeout，而非等待流消费完成

代理服务器可能不支持 abort signal 的传播方式，导致请求在发送任何数据前就被终止。

## 修复

移除 AbortController，恢复原始 `stream()` 调用：
```typescript
const rawStream = await withRetry(() =>
  Promise.resolve(this.client.messages.stream(
    createParams as unknown as Anthropic.MessageStreamParams
  ))
);
```

超时由代理服务器自身处理（代理有自己的超时机制）。

- curl 测试 API: ✅ 流式响应正常返回
